# 毕设项目进度汇报：基于多模态大模型的智能教学系统
**汇报日期：2026-02-27**
**当前阶段：系统核心功能打通与多模态交互优化**

---

## 一、 已完成工作总结

### 1. 后端架构与 AI 引擎开发
*   **混合解析引擎实现**：开发了基于 Python (pdfplumber) 的高精度 PDF 解析模块，解决了 Go 原生库在处理复杂中文排版时的乱码问题，实现了教材内容的结构化提取。
*   **RAG (检索增强生成) 模块**：
    *   完成了基于 **Qdrant 向量数据库** 的语义检索系统。
    *   实现了“教材知识点自动切片 -> 向量化存储 -> 语义相关性检索”的完整流程。
    *   设计了**降级检索机制**：当向量数据库连接异常时，系统可自动回退至文本搜索，保证了 AI 响应的鲁棒性。
*   **多模态协议深度适配**：完成了对**豆包 (Doubao) Ark V3** 接口的集成，针对多模态 Responses API 进行了私有协议封装，支持图片与文本的混合输入。

### 2. 前端教学交互中心
*   **沉浸式学习界面**：基于 React 19 构建了集 PDF 阅读、结构化文稿查看、视频播放于一体的资源中心。
*   **视觉交互创新（视觉提问）**：
    *   实现了视频播放器的**实时截帧**功能。
    *   **优化算法**：在前端对截取画面进行 JPEG 压缩与质量控制（质量 0.5），将数据体积减小 90%，解决了多模态数据传输中的性能瓶颈。
*   **AI 智能侧边栏**：支持学生针对特定知识点提问，并能结合视频画面进行实时分析。

### 3. 工程化与容器化部署
*   **全栈容器化**：基于 Docker Compose 实现了 Go 后端、React 前端、MySQL、MongoDB 及 Qdrant 的一键协同启动。
*   **高性能网关配置**：配置 Nginx 作为反向代理，针对 AI 深度搜索的长连接需求，调优了 `proxy_read_timeout`（延长至 300s）和上传限制（100M）。

---

## 二、 系统当前状态
目前系统已经实现了**“教材看得到、视频能分析、AI 懂课本”**的阶段性目标。学生在观看视频教学时，若对屏幕上的某个图表或公式不理解，可以直接“截图提问”，AI 会结合教材库中的背景知识给出详细解答。

---

## 三、 下一阶段计划

### 1. 深度学习体验优化 (Short-term)
*   **Markdown 渲染集成**：为 AI 回答增加公式、代码块和图表的渲染支持，提升学习资料的可读性。
*   **RAG 性能调优**：优化文本分段策略，引入重排序 (Rerank) 机制提升检索精度。

### 2. 知识图谱自动化构建 (Mid-term)
*   **实体关系抽取**：利用大模型从教材中提取概念、原理及其相互关联。
*   **图数据库集成**：引入 Neo4j 存储教学知识图谱，实现知识体系的可视化展示。

### 3. 教学质量评价模块 (Long-term)
*   **学习数据画像**：记录并分析学生的提问频率与关注点，为教师提供学情分析报告。
*   **系统安全性增强**：完善 JWT 权限校验与 API 限流，确保系统的生产级安全性。
